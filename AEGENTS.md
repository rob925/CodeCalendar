# Заметки по проекту CodeCalendar

## Назначение

CodeCalendar — статический сайт для управления событиями по программированию и IT. Идея похожа на компактный Google Calendar для технологических событий: пользователь смотрит события по месяцам, фильтрует по категориям, открывает подробности, переходит по ссылке-источнику и регистрируется/отменяет регистрацию. Регистрации сохраняются в браузере через `localStorage`.

Сайт поддерживает:

- русский и английский интерфейс;
- светлую и темную темы;
- категории: хакатоны, олимпиады, AI, конференции, воркшопы, митапы;
- календарь, список событий, модальное окно подробностей и режим «Мои события».

## Файлы

- `index.html`
  Основная разметка: оболочка приложения, шапка, первый экран, панель управления, календарь, список событий и модальное окно.

- `styles.css`
  Весь дизайн: переменные тем, shader-style фон, сетки, адаптивность, ячейки календаря, карточки событий и модальное окно.

- `app.js`
  Все данные и логика: переводы, категории, события, состояние, рендеринг, фильтры, регистрация, переключение темы/языка и открытие подробностей.

## Модель данных

Все редактируемые данные событий находятся в `app.js`.

Переводы событий:

- `translations.ru.events`
- `translations.en.events`

Каждый `id` события должен быть в обоих языках.

Записи календаря находятся в:

```js
const events = [
  { id: "event-id", category: "ai", date: "2026-07-03", url: "https://example.com" }
];
```

`id` должен совпадать с ключом в переводах. `category` должен совпадать с одной из категорий:

```js
const categories = [
  { id: "hackathon", color: "#0f766e" },
  { id: "olympiad", color: "#c2410c" },
  { id: "ai", color: "#7c3aed" },
  { id: "conference", color: "#2563eb" },
  { id: "workshop", color: "#9333ea" },
  { id: "meetup", color: "#be123c" }
];
```

Если добавляешь новую категорию, обнови:

- `categories`;
- `translations.ru.categories`;
- `translations.en.categories`.

## Важные функции

- `render()`
  Перерисовывает весь интерфейс.

- `renderFilters()`
  Создает кнопки-фильтры категорий.

- `renderCalendar()`
  Создает сетку месяца и события внутри дней.

- `renderEventList()`
  Создает правый список отфильтрованных событий.

- `openEvent(id)`
  Открывает модальное окно события и заполняет подробности.

- `toggleRegistration()`
  Добавляет/удаляет активное событие из `localStorage`.

- `dateKey(date)`
  Делает локальный ключ `YYYY-MM-DD`. Не удалять: защищает календарь от сдвига дат из-за часовых поясов.

## Частые изменения

### Добавить событие

1. Добавить русский текст в `translations.ru.events`.
2. Добавить английский текст в `translations.en.events`.
3. Добавить дату, категорию и ссылку в `const events`.
4. Запустить `node --check app.js`.

### Изменить главный текст

Редактировать:

- `translations.ru.heroTitle`;
- `translations.en.heroTitle`;
- при необходимости `translations.ru.heroSubtitle`;
- при необходимости `translations.en.heroSubtitle`.

Размер и вид главного заголовка: `styles.css`, селектор `.hero h1`.

### Изменить цвета темы

Редактировать CSS-переменные в начале `styles.css`:

- `:root` — светлая тема;
- `[data-theme="dark"]` — темная тема.

### Изменить shader-фон

Редактировать в `styles.css`:

- `.shader-bg`;
- `.shader-layer-a`;
- `.shader-layer-b`;
- `.shader-grid`.

### Изменить layout

Основные селекторы:

- `.app-shell`;
- `.hero`;
- `.control-band`;
- `.workspace`;
- `.calendar-wrap`;
- `.event-list-panel`.

Адаптивные правила находятся внизу `styles.css`.

## Мобильная версия

Мобильная адаптация сделана только CSS-ом, без изменения логики в `app.js`.

Основные брейкпоинты:

- `@media (max-width: 1120px)`
  Переводит главный экран и рабочую область из двух колонок в одну. Блок статистики `.hero-panel` становится горизонтальной сеткой из трех колонок.

- `@media (max-width: 820px)`
  Базовый мобильный/планшетный режим. Шапка `.topbar` и панель `.control-band` складываются вертикально, `.toolbar` становится сеткой с переключателем языка и кнопкой темы, фильтры `.filters` превращаются в горизонтальную прокрутку. Для длинного текста включен `overflow-wrap: anywhere`, чтобы заголовки и подписи не распирали экран.

- `@media (max-width: 640px)`
  Телефонный режим. Уменьшаются отступы, заголовки, статистика, карточки и модальное окно. Календарь остается сеткой из 7 дней без горизонтального скролла.

Особенности календаря на телефоне:

- `.weekday-row` и `.calendar-grid` получают `grid-template-columns: repeat(7, minmax(0, 1fr))`;
- `.calendar-cell` становится компактной ячейкой высотой около `66px`;
- `.event-chip` превращается в цветной круглый маркер `10px`, а текст внутри скрывается визуально;
- подробности события остаются доступны через клик по маркеру и через список событий ниже календаря;
- зарегистрированные события в календаре помечаются обводкой через `.event-chip.is-registered`.

Особенности фильтров:

- на мобильных фильтры не переносятся на много строк;
- используется горизонтальная прокрутка у `.filters`;
- скроллбар скрыт через `scrollbar-width: none` и `::-webkit-scrollbar`.

Особенности модального окна:

- `.event-dialog` на телефоне занимает почти всю ширину экрана;
- `.dialog-actions` складываются в колонку;
- `.primary-button` и `.primary-link` становятся кнопками на всю ширину;
- `.details-grid` уже на `820px` становится одной колонкой.

Если меняешь мобильную версию, особенно проверь:

- нет ли горизонтального переполнения страницы;
- помещаются ли длинные русские заголовки;
- не уезжает ли кнопка темы в `.toolbar`;
- видны ли все 7 дней календаря на ширине около `390px`;
- открывается ли событие по маленькому маркеру в календаре;
- удобно ли читать список событий под календарем.

## Проверка

Проверить синтаксис JS:

```powershell
node --check app.js
```

Проверить, что у каждого события есть перевод:

```powershell
@'
const fs = require('fs');
const src = fs.readFileSync('app.js', 'utf8');
const eventsBlock = src.slice(src.indexOf('const events = ['), src.indexOf('];', src.indexOf('const events = [')));
const ids = [...eventsBlock.matchAll(/\{ id: "([^"]+)"/g)].map(m => m[1]);
const missing = ids.filter(id => !src.includes(`"${id}": {`));
console.log(missing.length ? `Missing translations: ${missing.join(', ')}` : `All ${ids.length} events have translation entries`);
'@ | node -
```

## Текущие заметки

- Проект статический, сборка не нужна.
- Для просмотра открыть `index.html` в браузере.
- Регистрации хранятся в `localStorage` под ключом `cc-registered`.
- Язык хранится под ключом `cc-lang`.
- Тема хранится под ключом `cc-theme`.
- Перед `git push` проверяй `git status --short --branch`; если локальная ветка отстает от `origin/main`, сначала делай `git pull --rebase`, затем повторяй `git push`.
- Маркер «сегодня» сейчас жестко задан как `2026-07-03` внутри `renderCalendar()`.
- Для олимпиад используются источники с `olimpiada.ru`.
- Для AI-событий используются источники `ai-confa.ru`, `networkly.app`, `aisummit.ru` и страница «Цифрового прорыва».
